## חלק א׳ — איך לתקשר עם המשתמש בצ׳אט (הכי חשוב)

המשתמש כותב בעברית ומעדיף תשובות בעברית. הבעיה המרכזית שלי: **ערבוב עברית ואנגלית באותו משפט מתחרבש ויזואלית** (כיווניות / BiDi) והמשפט נהיה מבולגן ולא ברור.

הכלל העליון: **כל שורה ומשפט בכיוון אחד בלבד.**

### מה גורם לבלגן (להימנע)

1. **תחילית עברית מודבקת למילה לטינית** — הכי גרוע. נהפך ויזואלית:
   - ❌ `ל-false`, `מ-remote`, `ב-main`, `ה-build`, `ל-RTL`
2. **מילה אנגלית/קוד באמצע משפט עברי**:
   - ❌ "שיניתי את ה-`images.unoptimized` והרצתי `next build` כדי לוודא"
3. **קישור Markdown באמצע משפט עברי**:
   - ❌ "הוספתי ל-[app/page.tsx](app/page.tsx) סקציה חדשה"
4. **אשכול מספרים+סוגריים+סימנים באמצע משפט עברי**:
   - ❌ "הכול עובד (P5+P7+P8) בבילד"

### איך לכתוב נכון

1. **טקסט הסבר — עברית טהורה.** בלי מילים אנגליות משובצות.
2. **מונחים טכניים, נתיבים, פקודות, קוד, שמות אנגליים** — לבודד אותם:
   - בבלוק קוד על שורה נפרדת, **או**
   - בסוף שורה אחרי נקודתיים, **או**
   - כפריט ברשימה (התווית הטכנית בתחילת/סוף הפריט, לא באמצע משפט עברי).
3. **להימנע מהדפוס תחילית-עברית + לטינית.** במקום זה: לכתוב את המונח בסוגריים בסוף, או לנסח מחדש.
   - ❌ "דחפתי ל-main"
   - ✅ "דחיפה בוצעה לענף הראשי." (ואם צריך לציין שם — בשורה/בלוק נפרד)
4. **מספרים פשוטים בעברית — בסדר** ("172 עמודים", "24 ליקויים"). אשכולות מורכבים — לבודד.
5. **אם שורה מאוד טכנית** (הרבה קוד/אנגלית) — לכתוב אותה **כולה באנגלית**, ולא חצי-חצי.

### דוגמאות — לפני ואחרי

```
❌ "הוספתי ל-app/page.tsx סקציית EEAT עם Person schema ודחפתי ל-main"

✅ "הוספתי סקציית EEAT לדף הבית, כולל סכמת Person, ודחפתי לענף הראשי."
   ואז, בשורה נפרדת אם צריך נתיב:
   הקובץ: app/page.tsx
```

```
❌ "הבילד עבר (172 עמודים) ו-tsc נקי, רוצה שאריץ next start?"

✅ "הבילד עבר — 172 עמודים, ללא שגיאות טיפוסים.
    רוצה שאפעיל את השרת המקומי?"
```

### עקרון על
כשיש ספק — **שורות עברית קצרות**, ופקודות/נתיבים/קוד בבלוקים נפרדים. עדיף מבנה (רשימות, כותרות, בלוקי קוד) על פני משפט ארוך שמערבב כיוונים.

## חלק ב׳ — כתיבת עברית בתוך הקוד (קבצי `.tsx` / `.ts`)

אתר **בדק בית Legal** — אתר שיווקי/SEO בעברית. כמעט כל הטקסט מוטמע כמחרוזות.

### גרשיים וגרש — לא ASCII

בתוך מילה עברית, להשתמש בתווים העבריים — לא ב-ASCII ולא ב-escaping:

| תפקיד | התו | Unicode | דוגמאות |
|-------|-----|---------|---------|
| גרשיים | `״` | U+05F4 | `ת״י`, `ממ״ד`, `חוו״ד`, `תשל״ג`, `ש״ח` |
| גרש | `׳` | U+05F3 | `מס׳`, `הר יונה ב׳`, `יח׳` |

```ts
// ❌  "בדיקת ממ\"ד לפי ת\"י 1142"      (escaping מכוער, שגוי)
// ✅  "בדיקת ממ״ד לפי ת״י 1142"        (תווים עבריים, בלי escaping)
```

### לא לערבב אות לטינית בתוך מילה עברית (typo)

טעות נפוצה שלי: `האקלim` במקום `האקלים`, `תמiד` במקום `תמיד`. לפני סיום עריכת תוכן עברי:

```bash
rg '[א-ת][a-zA-Z]|[a-zA-Z][א-ת]' <file>
```

(שמות מותג כמו `YAMA TLV` לגיטימיים; מילה עברית עם אות לטינית — תמיד באג.)

### כיווניות
מכולות עם תוכן עברי צריכות `dir="rtl"`. רוב התבניות כבר עושות זאת.
